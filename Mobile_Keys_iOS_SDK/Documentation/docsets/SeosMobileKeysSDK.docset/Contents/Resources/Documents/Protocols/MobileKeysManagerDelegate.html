  • -mobileKeysDidTerminateEndpoint

    Callback for when the endpoint has been terminated. This can happen even without calling -[MobileKeysManager updateEndpoint]. From version 6.0.0 this will also be called if the SDK detects a fatal error with the local Seos Vault. The expected behaviour when this callback is made is to restart personalization.

    since version 2.0
